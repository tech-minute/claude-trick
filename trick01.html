01 — เทคนิคเจ๋งๆ ที่คนอื่นทำ
🪞 Meta Prompting

ให้ Claude เขียน Prompt ให้ตัวเอง

แทนที่จะเขียน prompt เอง ให้บอกงานกว้างๆ แล้วขอให้ Claude "เขียน prompt ที่ดีที่สุดสำหรับงานนี้" ก่อน แล้วค่อย run prompt นั้น

"เขียน prompt ที่จะทำให้ AI วิเคราะห์ financial statement ได้ดีที่สุด"
🎭 Persona

สวม Role เฉพาะทางสุดขีด

ไม่ใช่แค่ "เป็นผู้เชี่ยวชาญ" แต่ระบุให้ละเอียด เช่น "เป็น CTO ที่เคย scale startup จาก 0→100M users ใน 18 เดือน"

"You are a battle-hardened SRE who has survived 3 major outages at FAANG companies"
🔁 Iteration Loop

ให้ Claude วิจารณ์งานตัวเอง

หลังได้ output แล้วให้บอก "วิจารณ์สิ่งที่เพิ่งทำ แล้วทำใหม่ให้ดีกว่าเดิม 3 จุด" ได้ผลลัพธ์ที่ refined มากขึ้นอย่างเห็นได้ชัด

"Rate your previous response 1-10, identify the 3 weakest parts, then redo it"
🏷️ XML Tags

ใช้ XML tags จัด structure prompt

วิธีที่ Anthropic เองแนะนำ ใส่ context ใน <context> ใส่งานใน <task> ใส่รูปแบบที่ต้องการใน <format> Claude จะอ่านได้แม่นยำขึ้นมาก

<context>...</context>
<task>...</task>
<format>...</format>
🧠 Chain-of-Thought

บังคับให้คิดก่อนตอบ

เพิ่ม "Think step by step before answering. Put your reasoning in <thinking> tags" ทำให้ Claude ไม่ด่วนสรุป โดยเฉพาะโจทย์ logic หรือ math

"Think out loud in <thinking> tags, then give your final answer in <answer> tags"
📐 Output Control

ควบคุม format output ด้วยตัวอย่าง

แทนการอธิบาย format ยาวๆ ให้ใส่ตัวอย่าง 2-3 ชิ้นตรงๆ (Few-shot) Claude จะ pattern-match ได้เป๊ะกว่าการอธิบายเป็นคำพูดมาก

Input: "Apple" → Output: "🍎 Apple (fruit, red)"
Input: "Tesla" → Output: ?
🪄 Steelman

ขอ "ฝ่ายตรงข้าม" ที่แข็งแกร่งที่สุด

แทนการถามว่า "ข้อเสียคืออะไร" ให้ถามว่า "เขียน steelman argument ที่แข็งแกร่งที่สุดที่คัดค้านไอเดียนี้" ได้มุมมองที่ลึกและจริงกว่ามาก

"Write the strongest possible steelman against [your idea]"
🎯 Constraint Forcing

ใส่ข้อจำกัดบ้าๆ เพื่อความคิดสร้างสรรค์

บอก "อธิบายไอเดียนี้ในแบบที่เด็ก 10 ขวบเข้าใจ" หรือ "ใน tweet เดียว" หรือ "โดยไม่ใช้คำว่า AI เลย" — ได้ output ที่ clear และ creative ขึ้นมาก

"Explain this using only words a 5-year-old would know. No jargon allowed."
🕵️ Rubber Duck

ใช้ Claude เป็น "คนฟัง" แก้ปัญหา

แทนการถามวิธีแก้ ให้อธิบายปัญหาทีละขั้นแล้วบอก "อย่าเพิ่งช่วย แค่ถามคำถามที่ทำให้ฉันคิดต่อ" — มักแก้ปัญหาได้ด้วยตัวเอง

"Don't solve this. Just ask me clarifying questions until I figure it out myself."
02 — Pro Tips ที่คนมักมองข้าม
01

เปิด Projects ใส่ System Prompt ถาวร

สร้าง Project แล้วใส่ "Custom Instructions" ครั้งเดียว Claude จะจำ context ทุก conversation ในนั้น — ไม่ต้องพิมพ์ซ้ำทุกครั้ง

Projects Feature
02

อัปโหลดไฟล์แล้วให้สรุปก่อนถาม

ใส่ไฟล์ PDF/doc แล้วพูดว่า "summarize this in 5 bullets, then wait for my questions" — ประหยัด token และได้ตอบที่ตรงจุดกว่า

File Upload
03

ขอ "3 versions" แทนการ retry

แทนการกด regenerate ซ้ำๆ ให้ขอ "ให้ 3 version ที่แตกต่างกัน: formal, casual, creative" แล้วเลือกหรือ blend เอง

Creative Trick
04

บอก "อย่าขอโทษ / อย่า hedge" ใน prompt

เพิ่มใน prompt ว่า "Be direct. Don't apologize or add disclaimers." Claude จะตอบตรงๆ ไม่อ้อมค้อม เหมาะกับงาน technical หรือ feedback

Directness
05

ใช้ Extended Thinking กับโจทย์ยากๆ

เปิด "Extended Thinking" mode (Pro+) สำหรับโจทย์ที่ต้องการ reasoning ลึก เช่น strategy, math, code review — Claude จะใช้เวลานานขึ้นแต่ผลลัพธ์ต่างกันมาก

Pro Feature
06

ส่ง URL ให้ Claude อ่านแทนการ copy-paste

วาง URL ลงใน chat แล้วบอกว่า "อ่านหน้านี้แล้ว..." Claude จะดึง content มาวิเคราะห์ได้เลย ไม่ต้อง copy text ทั้งหน้า

03 — Prompt Cheat Sheet · สูตรสำเร็จ
Pattern วิธีใช้ เหมาะกับ
Act as [role] ให้ Claude สวม role เฉพาะทาง เช่น "Act as a senior data scientist" เพื่อรับมุมมองจากผู้เชี่ยวชาญ ขอความเห็น / วิเคราะห์เชิงลึก
Think step by step บังคับให้แสดง reasoning ทีละขั้น ไม่ด่วนสรุป ลดโอกาสผิดพลาดใน logic Math, logic, code debugging
Here are examples: [A→B, C→D] ให้ตัวอย่าง input/output 2-3 ชิ้นก่อน แล้วถามของจริง Claude จะ pattern-match ได้เป๊ะ สร้าง content ที่ต้องการ format แน่นอน
Critique then improve ให้ Claude วิจารณ์งานตัวเองก่อน แล้วทำ version ใหม่ที่แก้จุดอ่อน ได้ผลลัพธ์ refined กว่ามาก Writing, code review, design critique
What would [person] say? "What would Elon Musk / a skeptical investor / your harshest critic say about this?" — ได้มุมมองที่หลากหลาย ทดสอบไอเดีย / business plan
Rubber duck debug "I'll explain my problem. Don't solve it — just ask questions until I figure it out" — ช่วยคิดแก้เองได้ Problem-solving, code bugs
Summarize then Q&A ให้สรุปก่อน แล้ว "Now I'll ask specific questions" — ประหยัด context window และได้ตอบที่ตรงจุด อ่านเอกสารยาว, research
Give 3 options "Give me 3 different approaches: conservative, balanced, and bold" — ไม่ต้อง retry หลายรอบ ตัดสินใจ, creative work
ELI5 / ELI-PhD Explain Like I'm 5 หรือ "explain assuming I have a PhD in this field" — ปรับระดับความลึก เรียนรู้หัวข้อใหม่
Format: [JSON/table/bullets] ระบุ output format ตรงๆ ท้าย prompt เช่น "Respond only as JSON. No explanation." — ง่ายต่อการนำไปใช้ Developer / automation tasks
